**프로젝트명:** Escor AI 에이전트 설득 프레임워크 문서 작성
**목적:** Escor의 AI 에이전트 영업 및 내부 정렬을 위한 설득 논리 베이스 문서

에이전트 시장에서 Escor가 왜 적합한 파트너인가를 설명하는 스토리라인. 컨설팅팀과 기술팀이 고객 앞에서 같은 언어로 이야기할 수 있도록 하는 공통 기반이며, 독자(고객/내부)에 따라 가공해서 쓰는 원본 재료다.

**세 가지 역할을 동시에 수행하는 문서:**

1. 내부 정렬 — 컨설팅팀과 기술팀의 공통 언어
2. 고객 설득 논리의 베이스 — 독자에 따라 가공해서 사용
3. 각 메이커의 방향 제시

---

### Escor 맥락

- **회사 구조:** 컨설팅 사업부 + AI 기술 역량 + 풀스택 인프라 구축 노하우
- **방법론 이름:** S-Core Context Framework
- **핵심 포지셔닝:** 도메인 이해 + 설계 역량 + 기술 실행의 세 가지를 동시에 갖춘 구조.
- **주요 타겟:** 금융, 공공, 제조 산업의 의사결정자

---

### 작업 시 유의사항

이 문서는 **설득 논리의 베이스**입니다. 최종 산출물이 아니라 가공의 재료입니다. Claude와 작업 시 아래를 항상 염두에 두세요.

- 문서 작성에 몰입하지 말고 전체 스토리라인 설득력을 우선 판단할 것
- 재료에 갇히지 말고 독자 관점에서 논리 흐름을 검토할 것
- 챕터 추가/재구성은 전체 흐름 검토 후 결정할 것

#### 2. 논리 보강이 필요한 작업

**[해외 선도 기업 사례 방어 전략]**

"Harvey·Abridge는 쟤네라서 된 거 아니야?"라는 반론에 대한 방어가 현재 없음. 두 가지 방향 검토 필요.

- **방향 A — 시장 현상으로 치환:** 특정 기업 사례 대신 "도메인 특화 AI가 범용 AI 대비 4배 ROI"(FDE 문서), "FDE 채용 800% 급증" 같은 시장 흐름으로 대체. 특정 기업의 성공이 아니라 시장이 검증한 방향으로 프레임 이동.
- **방향 B — 패턴으로 서술:** 사례를 "저 회사의 성공"이 아니라 "이 패턴을 따른 곳들의 공통점"으로 재서술. 2장에서 이미 일부 적용됨. 강화 가능.

→ 두 방향 병용 가능. 의사결정 필요.

**[국내/해외 지표 활용 원칙 정립]**

무조건 국내가 좋은 것도, 해외가 나쁜 것도 아님. 아래 기준으로 정리 제안.

- **원칙 증명** → 해외 지표·사례 유지 (Harvey, BCG, RAND). 보편성이 필요한 자리.
- **현실 공감** → 국내 지표 우선 (대한상공회의소, 제조업 80% 실패). 독자가 "우리 얘기"로 읽는 자리.
- **사례 증거** → Escor 실제 사례로 교체 예정. 그 전까지 국내 유사 사례(신한은행 AI ONE, 마키나락스) 활용 검토.

#### 3. 방법론 세분화 — 어펜딕스 분리

현재 어펜딕스 A(S-Core 실행 가이드)가 하나의 문서에 모든 것을 담고 있음. 독자별 필요가 다름.

검토할 분리 구조:

|어펜딕스|내용|주요 독자|
|---|---|---|
|A. S-Core 실행 가이드|4단계 방법론, 이터레이션 운영|컨설턴트·엔지니어|
|B. 산업별 설계 가이드|금융·공공·제조 설계 원칙|컨설턴트|
|C. 기술 스펙|다섯 가지 전제조건, 스캐폴딩 원리, 방어적 설계|엔지니어·기술 검토자|
|D. Escor 조직 소개|구조, 역할, 프로세스, S-Core와의 연결|고객·의사결정자|

→ D가 현재 완전히 없음. "도대체 Escor가 무슨 기술을 가지고 있길래"에 답하는 문서. Escor 내부에서 채워야 하는 자리.

**방법론 시각화:**

- S-Core 4단계 순환 다이어그램 필요 (텍스트 기반 현재 구조를 시각 자료로)
- 이터레이션 복리 구조 시각화
- 결정론적/추론적 구역 이분화 시각화 → 별도 시각화 작업 필요. 완성 후 어펜딕스 삽입.

#### 4. 포지셔닝 구체화

현재 5장이 "세 역량의 조합이 필요하고 드물다"로 끝남. "그래서 Escor를 선택해야 하는 이유"가 명시적으로 없음.

필요한 작업:

- 시장 내 포지셔닝 맵 작성 (컨설팅/기술사/플랫폼/FDE 전문사 vs Escor)
- Escor와 함께하면 구체적으로 무엇을 얻는가 — 현재 7장에 일부 있지만 더 직접적인 언어 필요
- "왜 Escor인가"를 5장 말미 또는 별도 섹션에서 명시 검토

→ Escor 실제 사례·레퍼런스 확보 후 강화 가능. 지금은 뼈대만.
