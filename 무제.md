**2. 문제**

- PROBLEM STATEMENT
- *연결되지 않은 데이터는 ~~현황~~이 아니라 수치에 불과합니다*
- 이슈 트래커 / 코드 저장소 / 보안 스캐너 / CI/CD 파이프라인
- 기능 명세와 일정 현황만 존재. 실제 개발 진행 상황인 코드와 연결되지 않아 진척률의 신뢰도 낮음.
- 수많은 커밋과 브랜치 존재. 어떤 요구사항을 위한 변경인지, 보안 검수는 통과했는지 파악 난해.
- 별도의 리포트로 제공되는 취약점. 현재 개발 중인 버전의 영향도 파악이 어려워 배포 직전에야 확인.
- 빌드 성공/실패 로그만 존재. 실패 시 원인이 코드 문제인지 환경 문제인지 파악하기 위해 역추적 필요.
- CONSEQUENCES OF FRAGMENTATION
- 여러 도구 전환 반복 / Context Switching
- 수작업 데이터 취합 / Manual Aggregation
- 현황 파악 및 판단 지연 / Late Decision
- 마감일 지연 · 생산성 저하 / Critical Business Impact

---

**3. 해결**

- SOLUTION
- 소프트웨어 개발 주기의 모든 시그널을 하나로 연결합니다
- 실시간 가시성
- 모든 파이프라인의 진행 상태를 지표화하여 한눈에 파악합니다.
- 작업 단위 연결: 서로 다른 도구의 데이터를 하나의 작업 흐름으로 묶어 전체 맥락을 제공합니다.
- 도메인 간 교차 분석
- 단일 도구로는 보이지 않는 숨겨진 패턴을 발견합니다.
- AI 자동 탐지: 위험, 병목 현상, 요구사항 격차 등을 실시간으로 분석하여 즉시 보고합니다.
- 사용자 친화적 인터페이스
- 복잡한 쿼리 없이 자연어로 데이터를 탐색합니다.
- 다목적 활용: 대시보드 조망, 대화형 질의 탐색, 자동 리포트 수신까지 모두 지원합니다.

**4. 대시보드**

- INTEGRATED DASHBOARD + AI
- 5개 영역의 현황을 하나의 포탈에서 조망
- 이슈·요구사항 / 코드 활동 / 코드 리뷰 / 코드 품질 / 빌드·배포
- 지표 1. 납기를 맞출 수 있는가?
- 분석 로직: Jira 진척도 × Bitbucket 커밋 수 × 코드 복잡도 → 계획 대비 실제 진행 교차 검증
- 지표 2. 어디서 막히고 있는가?
- 분석 로직: 이슈 체류 시간 × PR 리뷰 사이클 × 커밋 활동성 → 병목 구간 자동 탐지
- 지표 3. 왜 이런 결과가 나왔는가?
- 분석 로직: 티켓 없는 코드 수정 × 코드 품질(Sonar) × 버그 발생 빈도 → 기술 부채 상관관계 파악
- 지표 4. 지금 배포 가능한가?
- 분석 로직: 보안 취약점(Sparrow) × 빌드 성공 여부 × 미완료 이슈 건수 → 릴리스 가능 여부 종합 판단
