# 에스코어 AI 에이전트 플랫폼 — 액션 아이템 v2

> 기준일: 2026년 3월 전제: 외부 매출 중심. 내부 프로젝트 없음.

---

## 확정된 전략 방향

|항목|결정 내용|
|---|---|
|런타임|OpenClaw 기반. 더 좋은 것 나오면 교체. 에이전트와 경쟁 안 함|
|CF|핵심 자산. 런타임 독립 표준 포맷으로 관리|
|MCP|이미 있는 것(더존, MS 등)은 가져다 씀. 없는 곳은 파트너십|
|포지션|크로스 시스템 오케스트레이터|
|버티컬 순서|코딩(현재) → 재무/금융컴플라이언스/공공(1순위) → 법무/HR(2순위)|

## 선행 과제 — 모든 것의 전제

### P-1. 버티컬별 첫 번째 외부 고객 특정 ★최우선

1순위 버티컬(재무·금융컴플라이언스·공공) 각각에 대해:

- 실제 구매 의향이 있는 외부 고객사 특정
- 고객사의 구체적 페인포인트 파악
- 이것이 확정되어야 Track B 개발 우선순위와 파트너십 대상이 결정됨

### P-2. CF 소유권 계약 조항 준비 ★최우선

- 불변 레이어(규제·표준): 에스코어 귀속
- 가변 레이어(고객사 고유 기준): 고객사 귀속
- 계약 전 정리 필수. 늦으면 협상력 없음

## Track A. 플랫폼 구축

### A-1. On-premise 배포 구조 ★최우선

- OpenClaw 기반 자체 서버 배포 패키지
- 데이터 외부 유출 없는 구조 검증
- **재무·금융 데이터 다루는 모든 버티컬의 선결 조건**

### A-2. CF 중앙 저장소 + 배포 파이프라인 ★최우선

- Skills / Rules / Knowledge 버전 관리
- CF 업데이트 시 전사 에이전트 동시 반영
- **재사용·지속성의 실체**

### A-3. 런타임 추상화 레이어

- CF를 특정 런타임 포맷에 종속되지 않도록 설계
- 내부 CF 표준 포맷 정의 → 런타임별 어댑터 변환
- 업계 표준(Anthropic Agent Skills, OpenClaw SKILL.md) 참조

### A-4. RBAC + SSO

- 역할 기반 접근 제어 (Admin / Developer / Auditor)
- 사내 계정 체계 연동 (LDAP/SAML)

### A-5. 감사 로그

- 전체 사용 이력 추적
- 금융·공공 컴플라이언스 대응 필수

### A-6. 관리 콘솔 UI

- 에이전트 등록·배포·모니터링
- OpenClaw WebChat 커스터마이징 또는 자체 구축

## Track B. 버티컬 에이전트 개발

> P-1(외부 고객 특정) 결과에 따라 우선순위 확정 아래는 선정 기준 기반 현재 가안

### B-0. 코딩 에이전트 CF 자산화 ★지금 당장

- 현재 프로젝트별로 흩어진 CF를 사내 라이브러리로 정리
- 불변 레이어(언어 공통 규칙)와 가변 레이어(고객사 표준) 분리
- 추가 개발 없이 지금 바로 시작 가능

### B-1. 재무/데이터 분석 에이전트

**형태**: 자연어 → SQL → 결과 해석 (Text-to-SQL)

CF 구성:

- Rules: 수치 출처 명시, DML 차단, 실행 전 쿼리 확인
- Skills: 이상치 드릴다운, 기간 비교 자동 포함
- Knowledge: 고객사 DB 스키마(RAG), KPI 정의 표준, 국내 회계기준
- Examples: 좋은/나쁜 SQL 변환 쌍

필요 MCP:

- DB 커넥터 (PostgreSQL, Oracle, MySQL)
- ERP 연동 (SAP MCP 활용, 더존 MCP 활용)
- 시각화 출력 (Excel, BI 도구)

**경쟁력**: 고객사 DB 스키마를 CF Knowledge로 구성하는 것이 진입장벽. 글로벌 솔루션은 고객사 내부 DB 구조를 모름.

### B-2. 국내 금융 컴플라이언스 에이전트

**형태**: 규제 문서 검토 + 체크리스트 자동화

CF 구성:

- Rules: 금융규제 필수 조항 체크리스트
- Knowledge: 금융감독원 규정, 내부통제 기준, 기존 납품 경험 기반 패턴
- Examples: 위험/안전 사례 비교

필요 MCP:

- 금융감독원 공시 API
- 내부 DMS 연동

**경쟁력**: 국내 금융규제 특화. 글로벌 벤더 공백 영역. 기존 금융사 납품 경험이 CF 불변 레이어로 전환 가능.

### B-3. 국내 공공/eGovFrame 에이전트

**형태**: 전자정부 표준 기반 개발 보조 + 행정 업무 자동화

CF 구성:

- Rules: eGovFrame 아키텍처 규칙, 행정 문서 표준
- Knowledge: 전자정부 프레임워크 문서, 행정 절차 DB

필요 MCP:

- 공공데이터포털 API
- 전자정부 연동 시스템

**경쟁력**: 국내 SI 경험 기반. 글로벌 벤더 진입 어려운 영역.

## Track C. 사업·파트너십

### C-1. 버티컬별 첫 번째 외부 고객 — 디스카운트 PoC

- 조건: 구축 비용 일부 디스카운트 ↔ 레퍼런스 공개 동의 + CF 불변 레이어 에스코어 귀속
- 타겟: AI 전담 조직 없는 중견 금융사, 공공기관, 규제 산업 기업

### C-2. MCP 파트너십 — 국내 레거시 시스템

MCP 없는 시스템 벤더와 공동 개발:

- 에스코어 제공: MCP 개발 + 에이전트 통합
- 상대방 제공: 고객사 채널 + 시스템 접근권
- 공동 레퍼런스로 양측 마케팅 활용

우선 접촉 대상:

- 공공기관 레거시 시스템 벤더
- 금융권 내부 시스템 벤더
- 국내 중견 ERP 벤더

### C-3. 도메인 전문기관 CF 공동 구축

CF의 핵심은 도메인 지식. 에스코어가 처음부터 만들기보다 협력이 빠름.

|버티컬|협력 대상|에스코어 제공|상대방 제공|
|---|---|---|---|
|재무/데이터|회계법인|기술·플랫폼|KPI 기준, 분석 패턴|
|금융 컴플라이언스|법무법인, 금융연구원|기술·플랫폼|규제 조문, 판례|
|공공|NIA(한국지능정보사회진흥원)|기술·플랫폼|전자정부 표준|

### C-4. 국내 데이터 벤더 파트너십

Anthropic이 FactSet, S&P Global과 파트너십 맺은 것처럼, 에스코어는 국내 데이터 벤더와 같은 구조 구축:

- 한국신용정보원
- 금융데이터거래소
- 공공데이터포털

```
지금 당장 (병렬 가능)
├── P-1. 버티컬별 외부 고객 특정       ← 모든 것의 전제
├── P-2. CF 소유권 계약 조항 준비

├── A-5. 감사 로그
```
