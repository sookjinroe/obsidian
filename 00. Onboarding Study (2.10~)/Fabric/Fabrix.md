# SDS Fabrix 2.0

### 소개

- 기업 내 데이터와 업무 시스템(ERP, CRM, SCM 등)을 생성형 AI와 연결하여 업무를 효율화하는 기업 전용 생성형 AI 서비스 플랫폼
- 커스텀 Agent의 개발/실행/관리를 모두 포괄하는 풀스택 플랫폼 지향(로드맵상)

### 벤치마크 대상

- AI 개발 플랫폼 (Platform)
    - 모델 선택, 튜닝, 학습, 배포, 모니터링까지 전체 라이프 사이클을 관리하는 인프라 통합 플랫폼  
        ex) 구글 vertex 같은 풀스택 개발 플랫폼들  
        _→ Fabrix는 기업 내부의 데이터와 레거시 시스템을 연결하는 개발 환경과 보안 인프라를 제공_
- AI 애플리케이션 (Application)
    - 특정 목적(검색, 요약, 이미지 생성 등)을 위해 모든 로직이 추상화된 최종 사용자용 AI 서비스  
        ex) 젠스파크, 퍼플렉시티 같은 딸깍 시스템  
        _→ Fabrix는 별도의 개발 없이 즉시 사용하는 Pre-built Agent와 반응형 UI 제공_

### 타겟

- 금융 · 공공 ·제조 업종을 주요 고객으로
- 엔드유저는 기업의 **일반 사용자**

### 고객 니즈

- 내 업무 프로세스 흐름에 맞게 설계, 통제 가능한 에이전트를 직접 만들고 싶어
- 직접 만든 에이전트를 반복 활용 가능하도록 에이전트를 표준화/자산화 하고 싶어
- 누구나 쓸만한 기능의 에이전트는 기본 제공되었으면 좋겠어(검색, OCR 등)

### 핵심 소구점

- AgentOps의 도입
    - 에이전트의 생성부터 배포, 거버넌스까지 아우르는 '운영 체계(Ops)'를 구축 → 서비스에서 플랫폼으로
- 동적 UI(Dynamic UI)
    - 텍스트 기반의 챗 UI를 벗어나 작업 특성(디자인, 코딩 등)에 맞는 UI를 생성 → 사용자 경험(UX) 개선
- Agentic Orchestration
    - 정적 워크플로우를 넘어 멀티에이전트 간의 자율 협업 체계 구현.

### 구성

- As-is(공홈 기준)
    - AI Lab: 업무에 특화된 Agent를 Self-Service로 제작, 배포하는 환경
    - Store: Agent와 프롬프트 공유하는 라이브러리
    - Connector: 내부 데이터와 업무 시스템, 외부 도구 등 연동
    - Chat: 에이전트와 대화하는 접점
- To-be(기술 로드맵 문서 기준)
    - Agentic App: 태스크에 맞는 동적 인터페이스(ex. 젠스파크)
    - Agent: 3가지 수행방식 → 태스크 특화(Unit)/단계형 워크플로우(Sequence)/멀티에이전트 협업(Orchestration)
    - Agent FrameWork: Agent가 동작하기 위한 계획∙탐색∙실행∙분석 수행 체계(Orchestrator+Memory)
    - Tool(Connector): 내/외부 모델, 데이터, 시스템 연동
    - AgentOps: 표준화된 개발 환경/실행 환경/거버넌스  
        **→ 전체적으로 쪼개고, 표준화해서 조립식으로 팔겠다는 취지**

### 제공 형태

- 모델 라인업
    - 삼성 Gauss3 모델(22B), On-Prem 환경 Gemini 3.0, 업스테이지 Solar
- 배포
    - SaaS, Private Cloud, On-Premise 모두 지원
- 운영
    - 망 분리 환경에서 독립 실행(Runtime)밎 동적 자원(GPU, CPU) 최적화를 관리하는 운영 인프라 제공

###   
오퍼링/가격 체계

- 구성 요소 단위로 상품 오퍼링을 재구성할 예정
- 리소스 기반 복합 과금
    - 구독료 (Subscription): 플랫폼 이용 기본료와
    - 사용량 (Usage): 사용자 수 + API 호출량 + Agent/Tool 실행 리소스 사용량 결합

### 참고1) '26년 FabriX 로드맵 요약

기존 제공하던 Fabrix를

- MSA 전환으로 기능별 독립 모듈화 하려고 함
- A2A, MCP, OpenAPI으로 에이전트 오케스트레이션을 표준화 하려고 함
- 에이전트의 기능(문서 작성, 디자인, 코딩 등)에 맞는 동적 UI를 구성하려고 함(기존은 고정 챗 UI)
- 정적 워크플로우에서 멀티에이전트 협업 구조로 전환하려고 함
- 통합 Pro/Low/No-code 개발 환경 및 개발 도구를 확보하려고 함(타겟 엔드 유저 확장 효과)
- Agent 실행 과정 추적 및 통합 대시보드 구축, 보안 관리 강화
- 신규 모델 추가, 답변 평가 체계로 품질 고도화
- 활용도가 높은 필수 기능 중심의 Unit Agent 다각화
