### 1. 문제 정의
"Context Framework"라는 용어의 해석이 모호하여 커뮤니케이션 미스의 가능성이 있음.
하나의 정의로 통일하여, 전략-기획-개발-컨설팅(+보고)를 관통하도록 정리하고자 함.
- 이 개념에 따라오픈소스 ↔ 자체개발의 경계가 될 수도 있음
	- 이는 우리의 IP, 경쟁력에 영향이 있을 수 있음
	- 제품의 서빙 형태나 운영에도 영향이 있을 수 있음

### 2. 논의 전제
**사업구조**
- **파는 것** — On-Premise AI 코딩 어시스턴트 (금융/공공 대상)
- **제품 구성** — 오픈소스 에이전트 + Context Framework (우리 IP)
- **오퍼링** — 제품 + 인프라 구축 + 운영
- **확장 방향** — SW 개발(SDLC) → Vertical Agent 생태계

**에이전트 방침** 
더 좋은 에이전트가 나오면 교체한다. 특정 에이전트에 종속되지 않는다.

**CF 경계 기준 — "agent가 런타임에 직접 쓰는가"**
- **포함** — agent가 런타임에 직접 참조하거나 실행하는 것들 (Tool, Knowledge, Rules, Skill 등 — 고정된 목록이 아니며 최적 조합으로 구성)
- **제외** — agent 외부에서 작동을 가능하게 해주는 것들 (인프라, 컨설팅)

### 3. 정의 후보
**안 A — 구조/설계 관점**
“소프트웨어/아키텍처적 의미의 프레임워크”
“코드/시스템 레벨의 구조적 개념”
"오픈소스 에이전트의 Framework를 활용하여 Extension Point를 특정 도메인에 맞게 채운 구조"
"우리가 프레임워크(Cline 등)를 써서 도메인 레이어를 구성했다

| 해석  | 내용                         |
| --- | -------------------------- |
| A-1 | Cline + 우리 레이어의 결합물 전체     |
| A-2 | 에이전트가 바뀌어도 유지되는 도메인 특화 레이어 |
| A-3 | 한번 정의되면 고정된 완성 패키지         |
| A-4 | 도메인 확장에 따라 지속적으로 진화하는 구조   |

**안 B — 구성요소 관점**
“개념적/사고적 의미의 프레임워크, 어떤 현상/문제를 바라보고 구성하는 논리적 틀, 코드가 없어도 성립하는 개념 구조"
"이것들이 왜 함께 묶였는지를 설명하는 체계로서의 프레임워크"
"특정 도메인에서 AI agent가 효과적으로 작동하기 위해 필요한 컨텍스트를 체계적으로 정의한 것"

| 해석  | 내용                                 |
| --- | ---------------------------------- |
| B-1 | 구성요소 리스트 자체. 이 문서가 곧 Framework     |
| B-2 | 구성요소를 만드는 원리와 기준. 문서는 결과물          |
| B-3 | SDLC에 한정된 개념                       |
| B-4 | SDLC는 첫 번째 인스턴스, 타 도메인에도 적용 가능한 개념 |

### 5. 두 안의 핵심 차이

| | 안 A | 안 B |
|---|---|---|
| 초점 | 어떻게 만들었는가 | 무엇으로 이루어졌는가 |
| 시선 | 공급자/개발자 | 사용자/기획자 |
| 에이전트 포함 여부 | 에이전트 내부에 CF 존재 | CF가 에이전트를 감쌈 |

**두 안 모두 에이전트 교체와 무관하게 개념은 유지됨**

### 7. 이해관계자 논의 자료

**경계 케이스 질문** — 정의를 들었을 때 "이건 포함되나요?"를 물어 각자의 해석을 드러냄

- 오픈소스 에이전트(Cline 등) 자체는 Context Framework인가?
- 인프라 설정(폐쇄망 런타임)은 Context Framework인가?
- 금융 가드레일 체크 Tool은 Context Framework인가?
- HR 에이전트에 적용한 컨텍스트 구조도 Context Framework인가?
- Context Framework는 고객마다 달라지는가, 동일한가?
- 에이전트가 교체되면 Context Framework도 바뀐 건가?

**역할극 질문** — 실제 업무 상황에서 어떻게 쓰는지를 물어 추상적 해석 차이를 구체화

- 고객사 제안 PT에서 Context Framework를 한 문장으로 설명한다면?
- 신규 팀원에게 "Context Framework 작업해줘"라고 했을 때 그 팀원이 뭘 해야 하는가?
- 경쟁사와 차별화를 설명할 때 Context Framework를 어떻게 포지셔닝하는가?
