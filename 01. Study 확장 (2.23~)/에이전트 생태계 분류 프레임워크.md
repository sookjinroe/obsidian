
본 프레임워크는 에이전트를 **대화 구조**, **실행 환경**, **구성 방식**의 3가지 축을 기준으로 분류합니다.

## 축 1. 대화 구조 (Conversation Structure)

사용자와 에이전트 간의 상호작용 방식 및 대화 흐름의 주도권을 기준으로 분류합니다.

| **유형**                 | **정의**                                            | **주요 사례**                                                                               |
| ---------------------- | ------------------------------------------------- | --------------------------------------------------------------------------------------- |
| **1-A. 오케스트레이터 위임형**   | 사용자는 오케스트레이터와만 소통하며, 내부 서브 에이전트의 작업 분배 및 통합은 은닉됨  | Claude Cowork, Airtable Superagent, Anthropic Research, MS Copilot Studio (Hub & Spoke) |
| **1-B. 자동 핸드오프형**      | 에이전트가 판단하여 적합한 전문 에이전트에게 대화를 자동 전환 (세션 및 히스토리 유지) | Agentforce, OpenAI Agents SDK, Copilot Studio 고객지원, Azure Logic Apps                    |
| **1-C. 사용자 주도 핸드오프형**  | 사용자가 UI 버튼이나 명령어를 통해 명시적으로 다음 단계 에이전트로 전환 결정      | VS Code Copilot Handoff (Plan→Implement→Review), GitHub Copilot Custom Agents           |
| **1-D. 사전 선택형**        | 대화 시작 전 사용자가 특정 역할, 지침, 도구가 설정된 에이전트를 직접 선택       | OpenAI GPTs, Claude Plugins, Cline (Rules/MCP), OpenCode                                |
| **1-E. 병렬 스웜(Swarm)형** | 중앙 통제 없이 에이전트 간 직접 통신 및 분산 작업 수행 (주로 백엔드 인프라 패턴)  | Amazon Nova Swarm, CrewAI, AutoGen, LangGraph (Multi-agent)                             |

## 축 2. 실행 환경 (Execution Environment)

에이전트가 작업을 수행하기 위해 접근하고 조작하는 환경을 기준으로 분류합니다.

- **2-A. API 커넥터:** 외부 서비스 API를 호출하여 데이터 수신 및 액션 실행 (MCP 표준 활용 증대)
    
    - _사례: Claude Cowork MCP, Cline MCP, Copilot Studio 커넥터_
        
- **2-B. 브라우저 조작:** 실제 브라우저를 구동하여 웹 탐색 및 폼 입력 등 사용자 행위 시뮬레이션
    
    - _사례: Manus, OpenAI Operator, Claude in Chrome, Hyperagent_
        
- **2-C. 로컬 파일시스템 / 샌드박스:** 로컬 PC 파일 접근 또는 격리된 VM/컨테이너 내 코드 실행
    
    - _사례: Claude Code, Cline(VS Code), Aider, OpenCode_
        
- **2-D. 코드 실행 샌드박스:** 파일시스템 접근 없이 격리된 환경에서 순수 연산(Python, Bash 등) 수행
    
    - _사례: ChatGPT Code Interpreter, Anthropic Research 서브에이전트_
        
- **2-E. 앱 내장 UI (양방향):** 기존 애플리케이션 내부에 에이전트가 탑재되어 앱 기능과 UI를 직접 공유
    
    - _사례: M365 Copilot (Word/Excel), Notion AI, Figma AI, Claude Add-ins_

## 축 3. 에이전트 구성 방식 (Configuration Method)

에이전트의 페르소나, 지침(Instruction), 도구가 패키징되는 기술적 방식을 기준으로 분류합니다.

- **3-A. 프롬프트 + 파일 기반:** 시스템 프롬프트 설정과 지식 파일 업로드 중심의 단순 구성
    
    - _사례: OpenAI GPTs, Poe Bots, Claude.ai 시스템 프롬프트_
        
- **3-B. 마크다운 파일 번들 기반:** 스킬, 커맨드, 커넥터 등을 `.md` 또는 `.json` 파일로 규격화하여 패키징
    
    - _사례: Claude Plugins(Cowork), VS Code Custom Agents (.agent.md), Claude Code_
        
- **3-C. 코드 기반 프레임워크:** SDK를 활용하여 로직, 툴 호출, 오케스트레이션을 프로그래밍 언어로 구현
    
    - _사례: LangGraph, CrewAI, AutoGen, OpenAI/Anthropic SDK_
        
- **3-D. 규칙 + MCP 조합 기반:** 에이전트 행동 규칙(Rules)과 MCP(Model Context Protocol) 서버 연결의 결합
    
    - _사례: Cline, OpenCode, Cursor, Windsurf_
        
- **3-E. 플랫폼 GUI 기반 (No-code):** 시각적 인터페이스(드래그 앤 드롭, 플로우차트)를 통한 워크플로우 설계
    
    - _사례: MS Copilot Studio, Salesforce Agentforce, n8n, Dify_
