## 1. Claude 애드인(Add-in)의 정의 및 핵심 UX
> **정의**: Microsoft Office 등 기존 호스트 앱 내부에 서드파티 기능을 삽입하는 확장 방식
- **작동 방식**: Excel/PowerPoint 실행 시 사이드패널에 Claude 채팅창 노출 및 파일 내용 읽기/쓰기 수행
- **핵심 UX**: **"파일을 보며 대화하기"** * 사용자가 Claude의 제안(예: "3열 합계 계산")과 실제 파일을 동시에 보며 실행 여부 판단 가능
    - 자연스러운 피드백과 허가 구조 형성
- **구조적 한계**:
     - 호스트 앱이 허용하는 범위 내에서만 작동
	- 단일 애드인으로는 서로 다른 앱(Excel ↔ PPT)을 넘나드는 멀티앱 작업 불가능

- **시나리오**: Excel에서 수익 데이터를 분석한 컨텍스트를 유지한 채, PowerPoint로 이동하여 요약 슬라이드 생성

## 3. Cowork 허가 시스템: 위임(Delegation) 모델
> 에이전트 방식에서는 사용자가 작업 과정을 실시간으로 보지 않으므로, 허가의 단위와 타이밍을 재설계함
- **작업 전 허가**: 특정 폴더 범위를 사전 지정하여 권한 위임
- **실행 모드 선택**:
    - `Ask`: 모든 액션마다 확인
    - `Code`: 파일 수정은 자율, 터미널 명령만 확인
    - `Full Auto`: 전체 자동 실행
- **가역성 기반 제어**: 삭제 등 되돌릴 수 없는(Irreversible) 작업은 실행 전 명시적 프롬프트 노출

### **[비교] 애드인 vs Cowork**

| **구분**     | **기존 애드인**            | **Cowork (에이전트)**      |
| ---------- | --------------------- | ---------------------- |
| **허가 단위**  | 개별 액션 단위              | 폴더 범위 + 체크포인트          |
| **사용자 역할** | 실시간 감독자               | 위임자 + 검토자              |
| **핵심 메타포** | "내가 하는 걸 Claude가 돕는다" | "Claude가 하는 걸 내가 위임한다" |

## 4. 에이전트 UI 지배적 패턴: 스플릿 스크린(Split Screen)
> Manus, Hyperagent, OpenAI Operator 등 주요 툴이 수렴하고 있는 표준 구조
- **레이아웃 구성**:
    - **[왼쪽 패널]**: 채팅 인터페이스, 지시 사항, 진행 로그 (친숙함 제공)
    - **[오른쪽 패널]**: 에이전트가 보는 화면, 파일 뷰어, 결과물 (책임감/관찰 가능성 제공)
- **의의**: 에이전트의 작업 과정을 실시간 관찰함으로써 신뢰(Accountability) 확보

## 5. UX 관점의 미해결 질문 (Future Task)
- **가시성 공백**: 현재 Claude 애드인은 앱 간 컨텍스트를 전달하지만, 사용자는 한 번에 하나의 앱(Excel 혹은 PPT) 화면만 점유함
- **비교 분석**: Manus 등은 에이전트가 앱 사이를 이동하는 과정을 한 화면에서 보여주지만, Claude 애드인 방식은 이 '이동 과정'을 실시간 관찰할 인터페이스가 아직 부재함
- **향후 과제**: Anthropic이 이 간극을 진행 로그 뷰어나 더 잦은 체크포인트 중 어떤 방식으로 채울지 관찰 필요
