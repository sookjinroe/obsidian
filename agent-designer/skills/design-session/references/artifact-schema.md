# 설계 산출물 형식 정의

## 단일 파일 구조

산출물은 `artifact.md` 하나로 관리한다. 단계가 진행되면서 컬럼이 추가된다. 파일을 새로 만들지 않는다.

| 단계 | 추가되는 컬럼 | 담당 |
|---|---|---|
| design-drafter 완료 | 발화ID, 유형, 발화내용, 요구사항ID, 요구사항 | design-drafter |
| plugin-dev 완료 | 구성요소명, 계위, 소속플러그인 | plugin-dev:plugin-structure |
| gap 검증 완료 | gap설명, 심각도 | design-session (gap-report 머지) |

---

## 전체 컬럼 정의

| 컬럼명 | 의미 | 추가 시점 |
|---|---|---|
| 발화 ID | 발화를 고유하게 참조하는 ID (U01, U02, ...) | drafter 완료 |
| 유형 | 일상 업무 / 임기응변 / 크로스 시스템 / 치명적 경로 / 보완 | drafter 완료 |
| 발화 내용 | 사용자가 실제로 할 말 | drafter 완료 |
| 요구사항 ID | 요구사항을 고유하게 참조하는 ID (R01, R02, ...) | drafter 완료 |
| 요구사항 내용 | 이 발화를 처리하기 위해 필요한 것 | drafter 완료 |
| 구성요소명 | 요구사항을 담당하는 컴포넌트 이름 | plugin-dev 완료 |
| 계위 | Command / Skill / references / MCP / Hook / Agent | plugin-dev 완료 |
| 소속 플러그인 | 이 구성요소가 속하는 플러그인 | plugin-dev 완료 |
| gap 설명 | 연결이 끊어진 지점과 이유 | gap 검증 완료 |
| 심각도 | 심각 / 중간 / 경미 / - | gap 검증 완료 |

## 1:N 관계 표현 규칙

발화 하나가 여러 요구사항에 대응될 때 발화 ID를 반복하는 행 구조로 표현한다.

**규칙:**
- 같은 발화 ID가 여러 행에 나타날 수 있다
- 같은 요구사항 ID가 여러 구성요소 행에 나타날 수 있다
- 발화 내용은 중복 행에서도 동일하게 반복한다 (역추적 가능성 유지)
