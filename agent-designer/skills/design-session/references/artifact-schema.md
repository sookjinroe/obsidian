# 설계 산출물 형식 정의

## 버전별 구조

산출물은 작업이 진행되면서 열이 점진적으로 추가된다.

| 버전 | 추가되는 정보 | 파일명 |
|---|---|---|
| v1 | 발화 ID, 유형, 발화 내용 | artifact-v1.md |
| v2 | 요구사항 ID, 요구사항 내용 | artifact-v2.md |
| v3 | 구성요소명, 계위, 소속 플러그인 | artifact-v3.md |
| v4 | (발화 행 추가됨, 기존 열 유지) | artifact-v4.md |
| v5 | gap 설명, 심각도 | artifact-v5.md |
| final | 확정 상태 (v5 기반) | artifact-final.md |

---

## 전체 컬럼 정의

| 컬럼명 | 의미 | 추가 시점 |
|---|---|---|
| 발화 ID | 발화를 고유하게 참조하는 ID (U01, U02, ...) | v1 |
| 유형 | 일상 업무 / 임기응변 / 크로스 시스템 / 치명적 경로 / 보완 | v1 |
| 발화 내용 | 사용자가 실제로 할 말 | v1 |
| 요구사항 ID | 요구사항을 고유하게 참조하는 ID (R01, R02, ...) | v2 |
| 요구사항 내용 | 이 발화를 처리하기 위해 필요한 것 | v2 |
| 구성요소명 | 요구사항을 담당하는 컴포넌트 이름 | v3 |
| 계위 | Command / Skill / references / MCP / Hook / Agent | v3 |
| 소속 플러그인 | 이 구성요소가 속하는 플러그인 | v3 |
| gap 설명 | 연결이 끊어진 지점과 이유 | v5 |
| 심각도 | 심각 / 중간 / 경미 / - | v5 |

## 1:N 관계 표현 규칙

발화 하나가 여러 요구사항에 대응되거나, 요구사항 하나가 여러 구성요소에 대응될 때, 발화 ID와 요구사항 ID를 반복하는 행 구조로 표현한다.

### 예시 (v3 기준)

```markdown
| 발화 ID | 유형 | 발화 내용 | 요구사항 ID | 요구사항 내용 | 구성요소명 | 계위 | 소속 플러그인 |
|--------|-----|---------|-----------|------------|---------|-----|------------|
| U01 | 일상 업무 | "3월 사업부별 매출 보여줘" | R01 | 자연어를 SQL로 변환할 수 있어야 한다 | sql-translator | Skill | bi-agent |
| U01 | 일상 업무 | "3월 사업부별 매출 보여줘" | R02 | DB 스키마를 참조할 수 있어야 한다 | schema-ref | references | bi-agent |
| U01 | 일상 업무 | "3월 사업부별 매출 보여줘" | R03 | DB에 읽기 전용으로 접근해야 한다 | db-connector | MCP | bi-agent |
| U01 | 일상 업무 | "3월 사업부별 매출 보여줘" | R04 | DML은 항상 차단해야 한다 | dml-guard | Hook | bi-agent |
| U02 | 임기응변 | "지난번 그거 다시 뽑아줘" | R05 | 이전 쿼리 이력을 참조할 수 있어야 한다 | query-history | Skill | bi-agent |
```

**규칙:**
- 같은 발화 ID가 여러 행에 나타날 수 있다
- 같은 요구사항 ID가 여러 구성요소 행에 나타날 수 있다 (1:N 요구사항-구성요소)
- 발화 내용은 중복 행에서도 동일하게 반복한다 (역추적 가능성 유지)

## v5 이후: gap 컬럼 추가

```markdown
| 발화 ID | 유형 | 발화 내용 | 요구사항 ID | 요구사항 내용 | 구성요소명 | 계위 | 소속 플러그인 | gap 설명 | 심각도 |
|--------|-----|---------|-----------|------------|---------|-----|------------|---------|------|
| U07 | 일상 업무 | "지난 분기 전체 법인 매출" | R11 | 법인 단위 집계 | (없음) | - | - | ①연결 없음: 법인 집계 구성요소 미존재 | 심각 |
| U02 | 임기응변 | "지난번 그거 다시 뽑아줘" | - | - | query-history | Skill | bi-agent | ③트리거 미정의 | 경미 |
```

gap이 없는 행은 gap 설명과 심각도를 `-`로 표시한다.
