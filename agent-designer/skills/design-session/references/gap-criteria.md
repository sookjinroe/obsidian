# gap 기준 및 심각도 분류

## 설계 시점 검증 기준 (작업 F, gap-analyzer 적용)

### ① 연결 없음

**정의:** 발화의 처리 경로에서 어떤 요구사항이 도출됐지만, 그 요구사항을 담당하는 구성요소가 artifact에 존재하지 않는다.

**판정 방법:**
- artifact-v2.md에서 해당 발화의 요구사항 체인을 확인
- 각 요구사항의 구성요소 열(artifact-v3.md~v4.md)이 비어 있거나 해당 구성요소명이 구성요소 목록에 없는 경우 → gap

**심각도:**

| 심각도 | 조건 |
|---|---|
| 심각 | 발화의 핵심 처리 단계(의도 파악, 데이터 접근, 처리)에서 연결이 끊어짐. 이 발화는 작동하지 않음 |
| 중간 | 발화가 부분적으로 작동하지만 결과가 불완전하거나 오류 가능성이 있음 |
| 경미 | 보조 기능(포맷 개선, 부가 정보 등)이 누락됨. 핵심 기능은 작동 |

**예시 — 심각:**
```
U07: "지난 분기 전체 법인 매출 합산해줘"
R11: "법인 단위 집계가 가능해야 한다"
구성요소: (없음) → ① 심각
```

### ③ 트리거 미정의

**정의:** artifact에 Skill 또는 Command가 존재하지만, artifact 전체에서 그것을 발동시키는 발화 경로가 정의되지 않았다.

**판정 방법:**
- 구성요소 목록에서 Skill과 Command를 추출
- 각 구성요소에 대해: artifact의 어느 발화도 이 구성요소로 연결되지 않는 경우 → gap
- 작업 E에서 커버리지 보완을 이미 했음에도 남아 있다면 더 심각하게 다룬다

**심각도:**

| 심각도 | 조건 |
|---|---|
| 심각 | 핵심 기능을 담당하는 Skill/Command에 발동 경로가 없음. 이 구성요소는 사실상 죽은 코드가 됨 |
| 중간 | 보조 기능을 담당하나 사용 빈도가 낮지 않을 것으로 예상 |
| 경미 | 보조 기능이며 사용 빈도가 낮거나 선택적 사용이 명확한 경우. Could 항목으로 이관 가능 |

**예시 — 경미:**
```
구성요소: export-report (Skill)
발화 연결: 없음 → ③ 경미 (보조 기능, Could 이관)
```

## 구현 이후 검증 기준 (requirements-sync 적용)

아래 세 기준은 구성요소 구현이 완료된 후에야 판정 가능하다. 설계 시점에는 적용하지 않는다.

### ② 입출력 불일치

앞 단계 구성요소의 출력 형식과 이 단계 구성요소의 입력 형식이 맞지 않는다.

### ④ 도구 미연결

이 Skill의 allowed-tools에 실제로 필요한 MCP 서버나 도구가 없다.

### ⑤ 실패 경로 미처리

이 구성요소가 실패할 때 에이전트의 행동이 정의되지 않았다.

## gap 처리 규칙

| 심각도 | 처리 방법 |
|---|---|
| 심각 | 즉시 작업 D로 복귀. 관련 요구사항과 구성요소를 재작성 |
| 중간 | 해당 발화만 재추적. 작업 D 복귀 여부는 영향 범위에 따라 판단 |
| 경미 | Could 항목으로 산출물에 기록. 작업 G 진행에 영향 없음 |
