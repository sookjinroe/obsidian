# 페르소나: 재무팀 데이터 분석가

## 역할 및 환경
- 직책/부서: 데이터 분석가 / 재무팀
- 주요 사용 시스템: Amazon Redshift, ⚠️ (BI 툴 미확인 — Tableau/Power BI/Looker 등 사용 가능성 있음), Excel 또는 Google Sheets
- 업무 루틴: 매출·비용·예산 데이터를 정기적으로 조회하여 보고서 작성, 경영진 요청에 따른 임시 데이터 추출, 월/분기 마감 작업 시 집중적으로 쿼리 수행

## 구조적 제약
- SQL 작성 역량이 있거나 없거나 무관하게, 쿼리 작성 자체가 반복·소모적인 작업으로 인식됨
- DML(INSERT / UPDATE / DELETE / DROP 등) 실행 권한이 없거나 조직 정책상 금지되어 있음
- ⚠️ Redshift 스키마 접근 범위(특정 스키마/테이블만 허용되는지 여부) 미확인

## 진짜 페인포인트
- SQL 문법 오류나 스키마 구조 파악에 시간을 소비하는 것이 본질적 업무가 아님
- "어떤 테이블에 어떤 컬럼이 있는지" 매번 확인하는 반복 작업
- 비슷한 쿼리를 조금씩 변형해서 재사용하는 패턴이 많아 자동화 여지가 큼
- 질문은 자연어로 이미 머릿속에 있는데 SQL로 '번역'하는 단계가 병목

## AI 활용 현황
- ⚠️ 현재 AI 툴 사용 여부 미확인 (ChatGPT로 SQL 생성 후 복붙하는 패턴 가능성 있음)

## 발화 스타일 특징
- 업무 도메인 약어 사용: "이번 달 매출", "전분기 대비", "YoY", "BU별", "코스트센터"
- 테이블명이나 컬럼명을 정확히 모르고 의미 중심으로 요청 ("매출 테이블에서...")
- 범위 조건을 구어체로 표현 ("이번 달", "지난 3개월", "작년 동기")
- 결과 형식을 지정하지 않는 경우가 많음 (암묵적으로 표 형태 기대)
