## 1. Executive Summary
<small> 왜 이게 필요한지, 뭘 하는 건지, 뭐가 좋아지는지 한눈에 정리 </small>

> [!note]+ Note
> 선행 프로젝트: 표준화 컨설팅 → "무엇을 해야 하는지"를 정의
> ㄴ 표준의 실행은 사람의 의지와 리소스에 달려 있음
>  ㄴ 실무자 반발: "산출물 분량을 기간 내에 맞추는 건 불가능에 가깝다"
>  선행 프로젝트 결과인 표준들이 실제로 작동하게 만드는 다음 단계를 후속 제안하기

**상황**
SW 개발 표준화 완성 → 그러나 표준이 실제로 지켜지는 구조는 아직 없음

**제안**
1차에서 만든 표준을 → AI 에이전트 실행의 기준으로 전환
개발 주기 전 구간에 에이전트를 배치하여 표준이 자동으로 작동하는 체계 구축

**기대 효과**
- 산출물 작성 부담 감소 — 에이전트가 표준 기반 초안 자동 생성
- 표준 준수 자동 모니터링 — 코드 커밋 시점에 검토 단계 트리거
- 납품 품질 가시화 — 코드 기반 역설계 문서로 설계 대비 구현 현황 확인


## 2. 현황 진단
<small>1차 과제의 결과물이 실제로 효용을 있으려면 어떤 어려움이 있을지 → 과제로 연결 브릿지</small>

> [!note]+ Note
>  프로젝트마다 협력업체가 바뀌거나 작업자가 바뀌는 경우, 표준을 가이드하는 비용이 반복됨 
> 또한 지속적인 표준 최적화가 필요한데, 개선 의지가 있어도 전파 비용이 크면 현실적으로 작동하기 어려움
> 표준의 실행 주체를 사람 → 시스템으로 전환 제안

**1차 프로젝트 성과**
- 개발 프로세스 표준화
- 단계별 산출물 및 양식 표준화
- 협력업체 관리 기준 수립

**표준 운영의 과제**

| 과제     | 내용                                       |
| ------ | ---------------------------------------- |
| 정보 비대칭 | 표준을 요구하는 주체(SK케미칼)와 실행하는 주체(협력업체)가 분리    |
| 실행 부담  | 워터폴 산출물 전량을 프로젝트 기간 내에 작성하는 것은 현실적으로 어려움 |
| 리소스 한계 | PM이 다수 프로젝트를 병행하며 산출물 전수 검토는 불가          |
| 표준 노화  | 표준 개선 시 재전파·재교육 비용으로 인해 초기 버전에서 멈추는 경향   |
표준의 실행 주체를 사람 → 시스템으로 전환할 필요성

## 3. 제안
<small>앞서 진단한 문제들을 해결하기 위한 구조 전환(사람 → AI 시스템) 제안, 구체적인 기능 설명 말고 핵심 개념과 차별점</small>

> [!note]+ Note
> 1차 표준 산출물이 각각 Framework의 구성 요소로 변환/적재되면, 
> 전파/교육이 필요한 문서가 아니라 에이전트가 자동으로 참조하는 기준이 됨
> 변환/적재 작업의 품질이 전체 시스템의 기반이 되며, 
> 1차를 수행한 우리가 하는 것이 외부 도구 도입하는 것보다 빠르고/품질이 좋을 것임을 어필

**제안의 핵심**
1차에서 만든 표준을 AI 에이전트의 실행 기준으로 전환하고, 
개발 주기 전 구간에 에이전트를 배치하여 표준이 자동으로 작동하는 체계를 구축

**두 가지 메커니즘**

| 구분                 | 역할                                                                     |
| ------------------ | ---------------------------------------------------------------------- |
| Context Framework  | SK케미칼 표준을 에이전트가 이해하는 형태로 적재<br>표준이 바뀌면 Framework만 업데이트 → 전 에이전트에 즉시 반영 |
| 개발 주기 전 구간 에이전트 배치 | 분석/기획 → 개발 → 커밋/PR → 빌드/테스트 → 검수 각 시점에 에이전트가 자동 개입                     |


**Why US**
- **1차 과제의 표준의 맥락 보유** 표준의 의도·예외·현장 맥락을 이미 보유
- S-Core만의 Context Framework 보유 — 표준을 에이전트 실행 기준으로 전환하는 기술 자산을 보유

## 4. 솔루션 구조
### 4-1. Context Framework
<small>context framework가 무엇인지, 단순 프롬프트 모음이 아니고 전체 인프라에서 어떤 역할의 레이어인지 소개</small>

> [!note]+ Note
> - Context Framework는 에이전트가 작업을 수행할 때 참조하는 모든 것의 집합, Rule / Skill / Prompt / MCP / 워크플로우 등
> 	- 표준/정책 텍스트를 담는 것 (Knowledge)
>	- 그걸 에이전트가 꺼내 쓸 수 있게 하는 것 (Retrieval, Context Scoping)
>	- 꺼낸 맥락을 상황에 맞게 추론하는 것 (Dynamic Synthesis)
>	- Context Window에 맞게 조립해서 주입하는 것
>	- 결과물이 표준에 맞는지 스스로 검증하는 것 (Validation)


**S-core Context Framework(SCF)란** 
범용 AI가 기업 고유의 업무 맥락을 이해하고 실행할 수 있도록하는 Knowledge와 Tool을 설계하는 레이어

**SCF가 담는 것**
- **Code Context**: Filesystem, Code Search
- **Knowledge Context**: Docs, Tech Spec, Wiki
- **Collaboration Context**: Jira, Git History, PR
- **Runtime Context**: Logs, Metrics, Test Result
→ 담긴 맥락을 에이전트가 실행할 수 있는 형태로 구조화 → Instruction, Skill, Rule, MCP 등 → 에이전트

### 4-2. SDLC 별 에이전트
<small>각 단계에서 에이전트가 구체적으로 무엇을 해준다는건지 대략적인 유저 시나리오</small>

> [!note]+ Note
> - 분석/기획: Plan Agent (Analyst → Architect → Test Designer)
>- 개발: Code Agent
>- 커밋/PR: Test Agent
>- 빌드/테스트: Test Agent
>- 검수: Tech Writer

![[Draft(세부 항목들 임의 작성함)-1772792015225.png]]

| 단계       | 에이전트        | 기능 예시                                                            |
| -------- | ----------- | ---------------------------------------------------------------- |
| 프로젝트 온보딩 | Code Agent  | 신규 투입 인력이 기존 코드베이스/표준/히스토리 파악                                    |
| 분석/기획    | Plan Agent  | RFP·요구사항 입력 → WBS, 요구사항정의서, 유스케이스 초안 자동 생성                       |
| 설계       | Plan Agent  | 화면/API/DB 설계서, 테스트 시나리오 초안 자동 생성                                 |
| 개발       | Code Agent  | 표준을 반영한 코드 제안 및 가이드 — 스캐폴딩, 네이밍/보안 실시간 가이드, 리팩토링 제안              |
| 커밋/PR    | Test Agent  | 코드 변경 시 자동 트리거 — 테스트 생성, 표준 준수 검토, CI/CD 파이프라인 구성, 결과 GitLab 코멘트 |
| 빌드/테스트   | Test Agent  | 빌드 실패 시 자동 트리거 — 로그 분석, 수정방안, Jira 이슈 자동 생성                      |
| 검수       | Tech Writer | 코드 → 설계문서 역생성, 설계 대비 구현 gap 분석                                   |
| 운영/유지보수  | Code Agent  | 장애 로그 분석, 레거시 현대화, 성능 이슈 탐지, 보안 정책 변경 반영                         |
