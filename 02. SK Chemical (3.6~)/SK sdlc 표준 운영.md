sdlc 프로세스 운영 체계
1. 착수: 표준 스택 확정
	- 기술 표준 카탈로그 + 기술 표준 가이드+프로젝트방법론 산출물
	- rfp 기술표준, 산출물 확정
	- 예외는 승인 필수
	- 개발/인프라 표준환경 제공
	- 사전품질검토(itsm)
	output: 표준 스택선정
2. 설계: 정의서, 가이드 기반 설계
	- 기술산출물 패키지 + 체크리스트 + 프로젝트방법론 산출물
	- 정의서/가이드 기반 설계
	- 설계 체크리스트 합의 검증
	- 중간 검수(선택)
	게이트: 설계 체크리스트
3. 개발: 코딩, 리뷰, ci/cd
	- 기술산출물 패키지 + 체크리스트 + 프로젝트방법론 산출물
	- 개발표준/가이드 준수
	- 형상관리, ci/cd 준수
	- 개발 체크리스트 합의 검증
	증빙: ci/cd, 개발, 품질 체크리스트
4. 테스트/배포: 검증, 릴리즈 통제
	- 기술산출물 패키지 + 체크리스트 + 프로젝트방법론 산출물
	- 단위/통합/성능 테스트
	- 표준, 품질, 보안 검증
	- 체크리스트 합의 검증
	- 사후 품질 검토(itsm)
	게이트: 테스트, 이행 체크리스트
5. 운영: 인수인계, 개선
	- 기술산출물 패키지 + 프로젝트방법론 산출물
	- 운영 인수/점검 합의 검증
	- 개선사항 개정 요청
	output: 최신버전 산출물 등록
