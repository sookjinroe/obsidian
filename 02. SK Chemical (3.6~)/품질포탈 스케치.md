- **1차 결과물**: 표준이 문서로 존재함
- **지금 문제**: 표준을 누가 어떻게 챙기냐 → PM은 인력이 부족하고, 회사는 전파가 안 됨
- **2차 목표**: 표준을 "시스템에 내재화"해서 사람이 직접 챙기지 않아도 돌아가게


**Push (에이전트)** → PM이 먼저 움직이지 않아도 됨. 임계치 초과, 산출물 미제출, 납기 D-7 같은 이벤트가 생기면 시스템이 먼저 알림. PM의 역할은 "순찰"이 아니라 "예외 처리"로 바뀜.

**Pull (대시보드)** → 보고나 게이트 리뷰 때 여러 도구를 뒤지지 않고 한 화면에서 전체 현황 확인. 숫자뿐 아니라 "이 프로젝트 지금 기준 대비 어떤 상태인가" 해석까지 포함.



**레이어 1 — 통합** Jira, Bitbucket, Sparrow, Confluence에 분산된 데이터를 한 곳에서 봄. 이것만으로도 도구를 넘나드는 수고가 없어짐.
산출물 품질 / 코드 품질 / 프로세스 품질 / 일정 품질을 한눈에

**레이어 2 — 자동 판단** 숫자를 보여주는 게 아니라, 1차에서 정의한 임계치와 자동으로 대조해서 "현재 기준 대비 어떤 상태인가"를 알려줌. PM이 기준을 외우거나 계산하지 않아도 됨.

**레이어 3 — AI 보조** 숫자로 표현 안 되는 것, 즉 산출물 내용이 표준을 충족하는지 — 이건 자동화가 불가능한 영역인데 AI가 체크리스트와 대조해서 초안 판단을 만들어줌. PM은 읽고 컨펌만 함.

**레이어 4 — 능동적 전달** PM이 들어와야 보이는 게 아니라, 시스템이 먼저 알림. 임계치 초과, 산출물 미제출, 납기 임박 같은 이벤트 발생 시 자동으로 푸시.


접점: 품질포탈
여러 도구에 흩어진 데이터를 한 곳에서 보여주고, SK케미칼 표준 임계치 기준으로 해석해주는 대시보드. 구체적인 지표 선정은 미정.
├── 대시보드
│   ├── 산출물 품질 → 제출 여부 + 내용 검수
│   ├── 코드 품질 → 시큐어코딩, unit test, 코드 커버리지, OSS취약점 등
│   ├── 프로세스 품질 → 코드 리뷰, CI/CD 파이프라인
│   └── 일정 품질 → 진척률, 병목, 이슈트래킹, 일정관리
│
└── 에이전트
    ├── 이벤트 트리거 (제출, 임계치 초과 등)
    ├── 일정 트리거 (납품 D-7 등)
    └── 산출물 작성 / 검수 보조

각 항목을 구체화해볼게요. "무엇을 보여주는가"와 "어떤 가치인가" 수준으로만 씁니다.

**AI가 하는 것의 범위**

자동 판단 가능한 것 — 숫자로 표현되는 지표는 임계치와 비교해서 pass/fail을 시스템이 냄

AI가 필요한 것 — 산출물 내용이 기준을 충족하는지는 텍스트 판단이 필요하므로 AI가 체크리스트 대조 후 초안 제시, 사람이 최종 판단

AI가 추가로 할 수 있는 것 — 여러 지표를 종합해서 "이 프로젝트 지금 어떤 상태인가"를 한 문장으로 요약. 숫자를 나열하는 게 아니라 맥락이 있는 해석.
