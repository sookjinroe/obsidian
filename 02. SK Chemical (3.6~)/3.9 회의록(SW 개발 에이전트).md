### 고객 관심사&페인포인트
- 표준 운영에 대한 부담, 산출물 작성에 대한 부담
- 최종 보고 후 아래 항목들에는 관심을 보임
	- 시큐어 코딩 / 코드 리뷰 / 템플릿 기반 산출물 생성 / 품질 포탈(△)
	- 위 기능에 대해서도, 개발자 레벨 기능을 원하는 건지, 검수용 기능을 원하는건지 모호함
	- 품질 포탈은 기존에 사용하는 도구들 있다고 함

### 3/9 논의내용
**제안 드래프트**
Context Framwork에 기술 표준 / 산출물 표준을 넣고
SW 개발 에이전트 공급해 표준 실행 주체를 사람에서 AI 시스템으로 전환 제안

**이슈**
1. 사용 주체: SW 개발 에이전트는 개발자 도구(IDE/CLI)임. SK PM이 직접 접근·사용할 용의가 있을지?
2. 산출물 작성 주체: 산출물 실 작성 주체는 수행사(개발자). 수행사의 생산성을 높여주는 도구를 SK케미칼이 구매할 인센티브가 불명확함.
3. 제품 형태: PM이 산출물을 직접 작성하는 경우가 적다면, 기능 대부분(산출물 초안 생성, 코드 가이드)이 PM에게 불필요해짐. 필요한 기능만 추려 PM용 UI로 감싸더라도, 실제 PM 니즈를 충분히 커버할 수 있을지 불확실함.
4. PM의 실제 니즈: "표준을 지키게 하는 것" 보다 "표준이 지켜졌는지 확인하는 것" 에 가깝지 않을지?








**그럼 PM에게 일반적으로 필요한 건?** → 검토/모니터링?
- 수행사가 제출한 산출물이 표준에 맞는지 검토
- 여러 프로젝트의 진행 상황 한눈에 확인 / 이슈만 확인
- 문제를 사전에 감지(납품 시점에 말고)

**우리 기능중 관심 가졌던 것을 PM 기준으로 해석하면?**
- 시큐어 코딩 → 시큐어 코딩 준수 여부 리포트
- 템플릿 기반 산출물 생성 → 산출물 관리 / 검수
- 코드리뷰 → ? 코드 레벨까지 검수하는지? 주체는?
- 품질포탈 → ? VSMP?

**기존에 쓰는 도구로 해결되는 범위는?(언급되었던 도구)**
- Jira: 이슈 트래킹, 진척 현황, 일정 관리
- Confluence: 문서 저장 및 공유, 표준 문서 보관
- Sparrow: 소스코드 보안 취약점 및 품질 결함 정적 분석 (보안약점, 코드스멜)
- Bitbucket: 코드 버전 관리, PR 이력

 **상용 도구로 안되는 범위는 있는지? → 제안해볼만한 것?**
- 산출물 내용 기준 검수(템플릿 검수 + 표준 반영 검수)
	- 입력 예: 수행사가 제출한 산출물 파일 + SK케미칼 표준 양식과 체크리스트
- 표준 준수 현황 요약
	- 입력 예: SonarQube 분석 결과 + GitLab PR/커밋 이력 + Jira 이슈 현황 + SK케미칼 기술표준/시큐어코딩 기준
	- 각 도구가 이미 개별 수치 제공, 통합도 DevLake 등 범용 도구로 커버?
- 설계 문서 역생성, 설계 대비 구현 gap 분석
	- 입력 예: 설계서(API 명세, DB 설계, 화면 설계) + 실제 코드베이스 또는 SonarQube/GitLab 분석 결과
- 시큐어 코딩 검수
	- 이미 사용중인 도구가 있는지? 어느레벨까지 검수필요한지?
