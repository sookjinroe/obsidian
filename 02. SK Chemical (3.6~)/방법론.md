
## 데이터 파이프라인

Jira·Bitbucket·Sparrow·Confluence·CI/CD 도구에서 REST API로 데이터를 수집하고, DevLake가 정규화된 DB로 통합한다. 포탈은 DevLake DB를 읽어 지표를 계산하고, SK케미칼 표준 임계치와 비교해서 상태를 판단한다.

## 품질 지표 분야 및 계산식

---

### 산출물 품질

**제출 완료율**

```
Confluence API → space 내 페이지 목록 (제목, 라벨, 생성일)
표준에서 정의한 단계별 산출물 목록과 대조
제출 완료율 = 페이지가 존재하는 산출물 수 / 해당 단계 전체 산출물 수
```

**기한 초과 여부**

```
Confluence 페이지 생성일 vs WBS에 정의된 제출 예정일 비교
생성일 > 예정일이면 기한 초과
```

**AI 검수 결과 (충족/미충족/불명확)**

```
Confluence API → 페이지 본문 추출
1차 표준 체크리스트 로드
LLM: 문서 본문과 체크리스트 항목을 대조하여 항목별 충족 여부 판단 + 근거 문장 생성
결과: 항목별 충족/미충족/불명확 + PM 컨펌 대기 상태로 큐 등록
```

### 코드 품질

**고위험/중위험/저위험 취약점 건수**

```
Sparrow export (JSON/XML) → 취약점 목록 (심각도, 상태: 신규/기존/해결)
심각도별 건수 집계
신규 발생 = 이전 분석 결과와 diff하여 새로 등장한 항목
기준값(고위험 0건, 중위험 N건 이하)과 비교 → 상태 판단
```

**취약점 추이**

```
스프린트별 Sparrow 분석 결과 이력 누적
심각도별 건수 시계열로 표시
```

**코드 커버리지**

```
CI/CD 파이프라인 빌드 결과 → 커버리지 리포트 수집
현재 수치를 표준 기준값(예: 80%)과 비교 → 상태 판단
스프린트별 수치 누적으로 추이 표시
```

**OSS 취약점**

```
OSS 분석 도구 → 취약점 목록 (심각도별)
중위험 기준값과 비교 → 상태 판단
```

**Sparrow 미실행 감지**

```
Sparrow 분석 실행 이력 → 마지막 실행 타임스탬프
현재 시각과의 차이가 N일 초과 → 프로세스 미준수 플래그
```

### 프로세스 품질

**리뷰 없는 머지**

```
Bitbucket API → PR 목록 (상태: merged, approval 목록)
approval이 0건인 상태로 merged된 PR 필터
건수 집계 → 표준 위반으로 플래그
```

**승인 미달 PR**

```
Bitbucket API → PR별 approval 수
approval 수 < 표준 기준(예: 2명) 인 merged PR 필터
```

**브랜치 준수율**

```
Bitbucket API → 브랜치 목록 (브랜치명)
표준 패턴(feature/, hotfix/, release/ 등) 정규식 매칭
준수율 = 패턴 매칭된 브랜치 수 / 전체 브랜치 수
```

**허위 보고 감지**

```
Jira API → Done 상태로 변경된 이슈 목록 + 이슈 ID
Bitbucket API → 이슈 ID가 포함된 커밋 이력
Jira 완료 처리 시각 기준 전후 N일 내 연관 커밋이 없으면 플래그
```

### 일정 품질

**WBS 진척률**

```
Jira API → 전체 이슈 목록 (상태별)
진척률 = Done 상태 이슈 수 / 전체 이슈 수 × 100
계획 진척률(WBS 기준)과 비교 → 상태 판단
```

**예상 납기 (velocity 기반)**

```
Jira API → 스프린트별 완료 story point 이력
velocity = 최근 N개 스프린트 완료 story point 평균
남은 story point / velocity = 예상 잔여 스프린트 수
예상 완료일 = 오늘 + 예상 잔여 스프린트 수 × 스프린트 기간
예상 완료일 > 납기일이면 지연 예상 + 초과일 수 표시
```

**병목 이슈**

```
Jira API → 이슈별 상태 변경 이력 (타임스탬프)
In Progress 상태 진입 시각 기준 현재까지 체류 일수 계산
체류 일수 > 기준값(예: 7일) 이면 병목으로 분류
```

**스프린트 완료율**

```
Jira API → 스프린트별 계획 이슈 수, 실제 완료 이슈 수
완료율 = 스프린트 내 Done 이슈 수 / 스프린트 시작 시점 계획 이슈 수
```

## 4. AI 교차 해석

각 영역의 지표는 독립적으로 계산되지만, AI는 영역을 가로질러 패턴을 읽는다.

```
입력: 4개 영역 지표 계산 결과 전체
LLM: 영역 간 상관관계 및 복합 리스크 패턴 해석
출력: 종합 요약 문장 (단순 상태 나열이 아닌 맥락 포함)
```

교차 해석 예시:

- 리뷰 없는 머지 증가 시점 ↔ 취약점 증가 시점 일치 → 프로세스 위반이 코드 품질 저하로 이어지는 패턴 감지
- 취약점 미해소 + 일정 지연 동시 발생 → 품질 개선을 일정 확보에 희생하고 있을 가능성
- 스프린트 완료율 저하 + 커버리지 상승 → 신규 개발보다 테스트 작업에 리소스 집중 신호
