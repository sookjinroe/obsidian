**1. 문제**


개발 활동 기록은 이슈 트래커, 코드 저장소, 보안 스캐너, CI/CD 파이프라인에 각각 저장됨. 
각 도구는 자기 영역의 데이터만 보여주며 연결되지 않은 데이터는 현황이 아니라 수치
현황을 파악하려면 여러 도구를 열고, 데이터를 취합하고, 직접 해석필요
마감일 지연, 생산성 저하 등에 영향


**2. 해결**
품질 포탈은 소프트웨어 개발 주기에서에서 발생하는 수많은 파편화된 시그널을 통합하여, 개발 주기의 예측 가능성과 운영 효율성을 높인다
품질 포탈은 개발 활동 데이터를 연결하고 AI로 관계를 해석해서 무엇이 잘못되었는지 파악
- 모든 개발 파이프라인의 실시간 진행 상태를 지표화해서 가시성 확보할 수 있고
	- 서로 다른 도구의 데이터를 작업 단위로 묶어주는 
- 도메인 간 교차 분석으로 단일 도구로는 보이지 않는 것을 발견할 수 있음
	- 에이전트는 별도의 지시 없이 지속적으로 실행되어 패턴이 나타나는 즉시 위험, 병목 현상, 요구 사항 격차 등을 파악해서 보고하는
- 사용자 친화적인 인터페이스
	- 대시보드 조망 / 질의로 탐색 / 자동 리포트 수신할 수 있는

---

**3. 통합 대시보드 + AI**
각 영역의 현황을 하나의 포탈에서 조망하여 소프트웨어 개발 프로세스에 대한 포괄적인 시각을 제공
근본 원인을 파악하고 실질적인 해답을 제시하는 ai

기본 현황: 이슈·요구사항/ 코드 활동 / 코드 리뷰 / 코드 품질 / 빌드·배포

지표1. 납기를 맞출 수 있는가?
   Jira 진척 × Bitbucket 커밋 ×코드 복잡도 — 계획 대비 실제 진행을 교차 검증

지표2. 어디서 막히고 있는가?
   이슈 체류 × PR 사이클 × 커밋 활동 — 병목 지점과 원인을 함께

지표3. 왜 이런 결과가 나왔는가?
   티켓 없이 진행되는 코드 수정 × 코드 품질 — 기술 부채의 상관관계 파악

지표4. 지금 상태로 배포 가능한가?
   취약점 × 빌드 × 미완료 이슈 — 릴리스 가능 여부를 종합 판단


**4. 탐색: 대화**

엔지니어링 데이터를 탐색. 미리 구성된 보기나 복잡한 필터 및 쿼리에 얽매이지 않고 AI 어시스턴트가 필요할 때 필요한 정보를 찾아줌

**대시보드 너머 엔지니어링 데이터에 관한 어떤 질문이든**  
“ 개발 속도가 왜 떨어졌지?”
“최근 배포 장애가 잦은데 원인이 뭐야”
"내일 배포할 건들 중에 놓친 리스크 있어? 배포해도 돼?"
“이번 스프린트 결과 보고서 작성해줘” 같은 자연스러운 대화

**액션 플랜**
- **병목 해소:** "A팀의 코드 리뷰가 평균 48시간 정체 중. 특정 모듈 담당자에게 리뷰 요청 필요"      
- **품질 가이드:** "최근 배포 실패의 80%가 특정 라이브러리 충돌에서 발생. 관련 테스트 케이스 강화 필요"
- **규제 준수:** “이번 배포에 개인정보 취급 로직이 포함되어 있으나, 보안 검수(Sparrow) 결과가 반영되지 않음. 승인 프로세스를 차단 필요”
→ 보고서 작성
→ Slack/메신저로 즉시 알림 발송 및 Jira 티켓 상태 자동 업데이트
