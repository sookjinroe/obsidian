**대시보드**

---

**산출물 품질**

현황
- SDLC 단계별 산출물 제출 현황 (요구사항정의서 제출됨 / 설계서 미제출 등)
- 제출된 산출물의 AI 검수 결과 (항목별 충족/미충족/검토필요)
- 산출물별 최종 수정일, 작성자

판단
- 현재 단계에서 제출되었어야 할 산출물 대비 실제 제출 비율
- AI 검수에서 미충족 항목이 있는 산출물 수

액션
- 미제출 산출물 목록 + 예정 제출일
- AI가 미충족으로 판단한 항목 → PM 컨펌 대기 상태로 큐에 올라옴

---

**코드 품질**

현황
- 시큐어코딩 취약점 건수 추이 (스프린트/주 단위 그래프)
- 코드 커버리지 현재 수치
- OSS 취약점 신규/해결/잔존 건수
- 마지막 Sparrow 분석 실행 시점

판단
- 각 지표가 1차 표준 임계치 대비 어떤 상태인지 (초과/근접/여유)
- 취약점이 늘고 있는지 줄고 있는지 방향성

액션
- 임계치 초과 항목 + 해소까지 필요한 잔여 건수
- Sparrow 미실행이 N일 이상이면 분석 요청 필요로 표시

---

**프로세스 품질**

현황
- PR 수, 머지 수, 평균 리뷰 대기 시간
- 리뷰어별 참여 현황
- 브랜치 생성/삭제 이력
- CI/CD 파이프라인 실행 이력

판단
- 리뷰 없이 머지된 PR 비율
- 승인자 수 기준 미달 PR 수
- 브랜치 네이밍 표준 준수율

액션
- 표준 위반 PR 목록 (리뷰 없음, 승인 미달 등) → 수행사에 소명 요청 가능
- CI/CD 미실행 구간이 있으면 사유 확인 필요로 표시

**일정 품질**

현황
- WBS 대비 현재 진척률
- 스프린트별 계획 대비 완료 이슈 수 추이
- 이슈 상태별 분포 (To Do / In Progress / Done)
- 평균 이슈 체류 시간

판단
- 현재 속도(velocity) 기준으로 납기 내 완료 가능 여부
- 오래 머물고 있는 이슈 — 병목 감지
- Jira는 완료인데 Git 커밋이 없는 이슈 — 허위 보고 가능성

액션
- 병목 이슈 목록 + 체류 일수
- 납기 위험 시 예상 지연일 표시

**에이전트**

이벤트 트리거
- 산출물 제출 감지 → AI 검수 자동 시작 → 결과 PM에게 푸시
- 임계치 초과 발생 (고위험 취약점 신규, 리뷰 없는 머지 등) → 즉시 알림
- Jira 완료 처리됐는데 커밋 없음 감지 → 확인 요청 알림

일정 트리거
- 게이트/납품 D-7 → 현재 상태 기준 통과 가능 여부 자동 리포트
- 산출물 제출 예정일 D-3 → 미제출 산출물 리마인드

산출물 작성/검수 보조
- 수행사: 산출물 작성 시 표준 체크리스트 참조 가능
- PM: 제출된 산출물에 대해 AI가 체크리스트 항목별 판단 초안 제시 → PM이 컨펌/반려
- 여러 지표를 종합해서 "이 프로젝트 현재 상태" 한 문장 요약 (보고 시 활용 가능)
