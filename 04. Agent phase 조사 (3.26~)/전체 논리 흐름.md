## 전체 논리 흐름

---

### 1단계 — 시장 인식 교정: "에이전트, 다들 한다는데 왜 안 되는가"

**주장:** 대부분의 기업이 에이전트를 도입하지만, 실제로 작동하는 경우는 극소수다.

**논리 구조:**

- 에이전트 도입은 이미 대세가 됐다 (시장 규모, 도입률)
- 그런데 실제 결과는 기대와 다르다 (실패율 데이터)
- 왜 실패하는가 → 모델이 나빠서가 아니다 → 설계가 없어서다
- 범용 에이전트(Copilot류)의 현실 (침투율, ROI 미달 데이터)

**빈 곳 체크:** ✅ 없음. 데이터 충분.

---

### 2단계 — 격차 정의: "SOTA 에이전트는 다르다"

**주장:** 지금 시장에 나와 있는 최전선 에이전트는 범용 도구와 질적으로 다른 결과를 낸다.

**논리 구조:**

- 에이전트 능력은 지난 2년간 폭발적으로 향상됐다 (벤치마크 추이)
- SOTA 에이전트가 실제 엔터프라이즈에서 낸 결과 (Harvey, Devin, Abridge 사례)
- 범용 에이전트와의 격차는 속도가 아니라 가능성의 차이다
- 그런데 SOTA도 그냥 가져다 쓰면 안 된다 → 설계 없이는 효과가 나지 않는다

**빈 곳 체크:** ⚠️ 하나 있음.

"SOTA도 설계 없이는 효과가 없다"는 주장을 뒷받침하는 **Escor의 직접 사례**가 아직 없습니다. 현재는 외부 데이터(Claude Code 헤비/라이트유저 격차, Devin 플레이북 유무 차이)로만 뒷받침되는 상태입니다. 나중에 Escor 레퍼런스로 채워야 할 자리입니다.

---

### 3단계 — 진입 장벽 설명: "왜 아무나 못 하는가"

**주장:** SOTA 에이전트를 제대로 쓰려면 세 가지가 동시에 필요하다. 대부분의 회사는 하나만 가지고 있다.

**논리 구조:**

- 도메인 지식 — 어떤 업무 문제를 에이전트로 풀어야 하는지 알아야 한다
- 설계 역량 — 에이전트가 어떻게 작동해야 하는지 구조화할 수 있어야 한다
- 기술 실행력 — 그것을 실제로 고객 환경에 배포할 수 있어야 한다

빌더 플랫폼은 세 번째만 줍니다. 범용 에이전트는 세 가지 모두 없습니다. 컨설팅 회사는 첫 번째만 있습니다.

**빈 곳 체크:** ⚠️ 큰 공백 하나 있음.

이 3가지 조건이 "왜 Escor만 가능한가"로 자연스럽게 연결되려면, **Escor가 세 가지를 모두 갖췄다는 근거**가 필요합니다. 컨설팅 이력이 도메인 지식을, S-Core Framework가 설계 역량을, 개발 조직이 기술 실행력을 증명합니다. 이 연결을 명시적으로 만들어야 합니다.

---

### 4단계 — Escor 포지셔닝: "우리는 이렇게 다르다"

**주장:** Escor는 고객 업무를 진단하고, S-Core Framework로 에이전트를 설계하고, 고객 환경에 구조째로 납품한다.

**논리 구조:**

- 우리가 하는 것: 컨설팅으로 문제를 정의 → S-Core로 설계 → 고객 환경에 배포
- 고객이 받는 것: 에이전트 하나가 아니라 업무에 맞게 설계된 에이전트 시스템
- 다른 회사와 다른 점: 모델을 연결하는 게 아니라 구조를 설계한다
- S-Core Framework가 그 설계를 가능하게 하는 Escor의 방법론

**빈 곳 체크:** ⚠️ 두 가지 있음.

첫째, S-Core Framework 자체에 대한 설명이 고객 언어로 번역되지 않았습니다. 지금은 기술 언어입니다. "커맨드, 훅, MCP 선언"이 고객에게는 "무엇"으로 들려야 하는지가 필요합니다.

둘째, 왜 구조째로 납품하는 게 좋은가에 대한 **고객 입장의 가치** 설명이 빠져 있습니다. 유지보수 가능성, 확장 가능성, 블랙박스가 아님 — 이걸 고객 언어로 만들어야 합니다.

---

### 5단계 — 효과: "고객이 실제로 경험하는 것"

**주장:** 제대로 설계된 에이전트는 효율이 아니라 가능성을 바꾼다.

**논리 구조:**

- 단순 시간 절약이 아니다 (이건 누구나 말한다)
- 이전에 불가능했던 업무가 가능해진다
- 판단의 질이 바뀐다
- 사람의 역할이 바뀐다

**빈 곳 체크:** ⚠️ 가장 큰 공백.

Harvey, Devin, Abridge의 사례는 있지만 모두 코딩·법률·의료 도메인입니다. **Escor가 실제로 다루는 업무 맥락에서의 효과 표현**이 아직 없습니다. 이건 레퍼런스가 생기기 전까지는 "잠정적 언어"로 채워두고 나중에 실제 사례로 교체해야 하는 자리입니다.

---

### 6단계 — 다음 단계: "어떻게 시작하는가"

**주장:** 전체 도입 전에 먼저 진단부터 시작할 수 있다.

**논리 구조:**

- 진단으로 시작 → 어떤 업무가 에이전트에 맞는지 파악
- 설계 → S-Core Framework 기반 에이전트 설계
- 배포 → 고객 환경에 구조째 납품
- 확장 → 첫 성공을 기반으로 범위 확대

**빈 곳 체크:** ⚠️ Palantir 부트캠프 모델과 유사한 **"빠른 가치 증명" 방식**이 있는지 확인 필요. 현재 Escor의 영업 프로세스가 이 흐름을 지원하는지도 검토 필요.

---

## 전체 흐름 요약

```
시장 실패 → 실패 원인(설계 부재) → SOTA는 다르다
→ 하지만 SOTA도 설계 없이는 안 된다
→ 세 가지가 동시에 필요하다
→ Escor는 세 가지를 모두 갖췄다
→ S-Core Framework가 그 증거다
→ 고객이 받는 것은 에이전트가 아니라 시스템이다
→ 그 결과는 효율이 아니라 가능성이다
→ 진단으로 시작할 수 있다
```

공백 요약:

- **3단계→4단계 연결**: Escor가 세 가지를 갖췄다는 증거 연결
- **4단계**: S-Core를 고객 언어로 번역
- **4단계**: 구조째 납품의 고객 가치
- **5단계**: Escor 맥락의 효과 표현 (레퍼런스 필요)
- **6단계**: 빠른 가치 증명 방식 유무

## 근거 후보 목록

### A. 시장 실패 데이터

- MIT NANDA(2025): GenAI 파일럿 **95%**가 손익 영향 제로
- RAND(2024): AI 프로젝트 **80% 이상**이 프로덕션 미달
- BCG(2024): **74%**가 AI로부터 실질적 가치 미창출
- Gartner: 2027년까지 에이전틱 AI 프로젝트 **40% 이상 취소** 전망
- IDC: 33개 파일럿 중 프로덕션 도달 **4개**(88% 실패)
- S&P Global: AI 이니셔티브 포기 기업 비율 **17% → 42%**(1년간)
- McKinsey: 전체 조직 **3분의 2**가 파일럿에 갇혀 있음
- MIT: 내부 자체 개발 성공률 **33%** vs 외부 솔루션 **67%**

### B. 범용 에이전트 실패

- Microsoft Copilot: 4억 5천만 구독자 중 유료 사용자 **3.3%**
- Gartner: IT 리더의 **5%만** Copilot 확대 배포 추진
- 직원의 **72%**가 Copilot을 일상 업무에 통합 못 함
- **57%**는 구현 후 빠르게 사용 감소
- SharePoint 사용자 중 에이전트 상호작용 비율 **0.1%**(주간)

### C. 실패 원인 분류

- BCG: 성공 공식 = 알고리즘 10% + 기술 20% + **사람·프로세스 70%**
- RAND: 가장 흔한 실패 원인 = **문제 정의 오류**
- MIT NANDA: "핵심 이슈는 AI 모델 품질이 아니라 **학습 격차**"
- Gartner: **전략적 실패**가 기술적 한계보다 지배적

### D. SOTA 능력 향상

- SWE-bench: 2023년 **1.96%** → 2025년 **80.9%** (2년간 40배)
- Stanford AI Index: 1년간 **16배** 향상(4.4% → 71.7%)
- SWE-bench Verified **80%** vs Pro Private **15~18%** (3~5배 격차)
- 같은 모델 + 다른 스캐폴딩 = **10포인트** 성능 차이

### E. SOTA 에이전트 실제 성과

**Devin:**

- Nubank 레거시 마이그레이션: **8~12배** 빠르게, **20배** 낮은 비용
- ETL 마이그레이션: 파일당 **3~4시간 vs 인간 30~40시간**
- PR 병합률: **34% → 67%** (2배)

**Harvey:**

- ARR **$1억**, AmLaw 100의 **42%** 고객
- 법률 업무 6개 중 **5개**에서 변호사 기준 초과
- 파워유저 월간 **36.9시간** 절약
- 범용 모델 대비 **97%** 선호율

**Abridge:**

- 의사 인지부하 **61%** 감소

- 의사 1인당 하루 **30분** 절약 + **18% 더 많은 환자** 진료

### F. Palantir 모델

- AIP 부트캠프 전환율 **약 75%**
